# Fix Citation Formatting in PDF Export

## Context

When exporting to PDF via pandoc, in-text citations like `[@Smith2020]` appear as raw text instead of formatted citations (e.g., "(Smith 2020)"). The bundled `zotero.lua` Lua filter (from Better BibTeX) only handles DOCX and ODT formats. In `zotero.lua` line 2063-2070, `config.format` is only set when `FORMAT` matches `odt` or `docx`. For PDF (LaTeX), `config.format` stays `nil`, so `Cite_collect` and `Cite_replace` both bail out, leaving citations unprocessed.

The fix: for PDF exports, use pandoc's built-in `--citeproc` engine with a bibliography fetched from Zotero, instead of the Lua filter.

### 2. Add `loadContentForExport()` to `DocumentManager.swift`

The current `loadContent()` reads from the `Content` table, which includes the app-generated bibliography section. For PDF, pandoc's `--citeproc` generates its own bibliography. For DOCX/ODT, Zotero manages the bibliography after refresh. To avoid duplicates, strip the app-generated bibliography from all exports.

Add a new method that assembles content from blocks, excluding bibliography blocks only (keep notes -- footnotes are document content):

```swift
func loadContentForExport() throws -> String? {
    guard let db = projectDatabase, let pid = projectId else { return nil }
    let blocks = try db.fetchBlocks(projectId: pid)
    let exportBlocks = blocks.filter { !$0.isBibliography }
    return BlockParser.assembleMarkdown(from: exportBlocks)
}
```

Reuses existing `fetchBlocks()` and `BlockParser.assembleMarkdown(from:)`.

```swift
private func extractCitekeys(from content: String) -> [String] {
    let pattern = #"(?:\[|; )@([^\],;\s]+)"#
    guard let regex = try? NSRegularExpression(pattern: pattern) else { return [] }
    let range = NSRange(content.startIndex..., in: content)
    return regex.matches(in: content, range: range).compactMap { match in
        guard let r = Range(match.range(at: 1), in: content) else { return nil }
        return String(content[r])
    }
}
```
